# Panics & Recover

Panics signal unrecoverable conditions (programming errors, invariant violations). Prefer returning errors for expected failures.

```go
package main

import "fmt"

func mustDivide(a, b int) int {
    if b == 0 {
        panic("division by zero")
    }
    return a / b
}

func safe(fn func()) {
    defer func() {
        if r := recover(); r != nil {
            fmt.Println("recovered:", r)
        }
    }()
    fn()
}

func main() {
    safe(func() { _ = mustDivide(10, 0) })
    fmt.Println("continue")
}
```

<Admonition variant="warning" title="Use panic sparingly">
Reserve `panic` for programmer errors or truly exceptional conditions. Returning `error` keeps control flow explicit and testable.
</Admonition>
